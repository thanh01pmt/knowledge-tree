# Security Policy

## Supported Versions

We issue security updates and validation fixes for the active versions of the Knowledge Tree repository:

| Version | Supported          |
| ------- | ------------------ |
| v2.x    | :white_check_mark: |
| v1.x    | :x:                |

---

## Reporting a Vulnerability

We take the security of our data pipelines, environment variables, Supabase cloud integrations, and automation scripts seriously.

If you discover a security vulnerability, please **do not** open a public GitHub issue. Instead, report it privately following these steps:

1. Email your report to the maintainers at `security@knowledgetree.org` or open a private security advisory on GitHub.
2. Include a detailed description of the vulnerability, steps to reproduce, and any proof-of-concept scripts.
3. Allow up to 48 hours for an initial response from the security team.

---

## Responsible Disclosure Policy

- Please give us reasonable time to investigate and fix any reported vulnerability before publicly disclosing it.
- Do not access or modify data belonging to other users or databases without authorization.
- We appreciate your efforts in keeping the Knowledge Tree ecosystem safe and secure!
